# Wiki Ingest / Query / Lint 규칙

## Ingest (저장 규칙)

### 파일명 규칙
- 한글 파일명 사용 (공백 대신 하이픈)
- 카테고리/_index.md 는 해당 카테고리 인덱스
- 예: wiki/분양/특별공급-가이드.md

### 프론트매터 (필수)
```yaml
---
updated: YYYY-MM-DD
source: 출처 (법령명, MCP 도구명 등)
---
```

### 내용 구조
1. 제목 (H1)
2. 근거 법령 또는 출처
3. 본문 (표, 목록 적극 활용)
4. 포인트 / 주의사항

---

## Query (조회 규칙)

1. `wiki/` 폴더 먼저 탐색
2. 관련 파일 없으면 MCP 도구 사용
3. 조회 결과는 반드시 wiki/에 저장 후 응답

---

## Lint (유효성 검사)

| 카테고리 | 갱신 주기 | 검사 항목 |
|---------|---------|---------|
| 법령/ | 3개월 | 개정 여부 |
| 분양/ | 분기 | 정책 변경 |
| 단지분석/ | 수시 | 입주·계약 현황 |
| 고객상담/FAQ | 월 1회 | 최신 문의 반영 |

### Lint 실행 방법
- `updated` 날짜 기준으로 갱신 필요 파일 목록 출력
- 법령 MCP로 최신 조문과 비교
