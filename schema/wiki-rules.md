# Wiki 문서 작성 규칙

## 파일 구조
- 각 카테고리 폴더에 `_index.md` (목차 역할)
- 개별 문서: 주제별로 분리

## 필수 헤더 (법령 문서)
```markdown
> 출처: korean-law-mcp 조회 결과 | 최종 확인: YYYY-MM-DD
```

## 필수 헤더 (단지분석 문서)
```markdown
> 분석일: YYYY-MM-DD | 상태: 초안/검토완료/확정
```

## 업데이트 원칙
- 법령 변경 시 즉시 업데이트 + 날짜 갱신
- 단지 청약 완료 후 결과 추가
- 새 FAQ 발생 시 고객상담/ 추가

## raw/ 폴더 사용
- PDF, CSV, Excel 등 원본 파일 저장
- .gitignore 처리되어 GitHub에 업로드되지 않음
- 파일명 규칙: `YYYYMMDD_단지명_문서종류.확장자`
